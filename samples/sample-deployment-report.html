⚙ How It Works

Update Rule — Triggers when an incident is updated and severity changes (High, Medium, Low, or Informational). The Phishing Triage Agent typically changes severity, so this catches most agent updates. The action adds your protected tags back.

Create Rule — Triggers when a new incident is created. Stamps your protected tags on every new incident.

Idempotent: Adding a tag that already exists is a no-op. If the tag was stripped, it gets re-added. Safe to run on every update.

⚠️ Limitations

  • Only protects the specific tags you configured — not dynamic.
  • Update rule fires on severity changes. If an update doesn't change severity, that specific update won't trigger tag restoration.
  • For full dynamic tag restoration (any tag, any trigger), use the Logic App approach.
