⚠️ Known Limitations

  • Data Retention: SecurityIncident table governed by workspace retention (default 90 days). Azure Activity Log capped at 90 days. Tags removed before the retention window cannot be recovered.
  • Lookback Window: This report analyzed only the last 30 day(s). Actual affected incidents may exceed what is shown.
  • Agent Attribution: The agent is identified via ModifiedBy patterns (e.g., SecurityCopilotAgent, Microsoft Defender XDR). If Microsoft changes the agent identity string, detection may miss events.
  • Tag Restoration Scope: Only tags specifically removed by the agent are restored. Tags intentionally removed by human analysts are not re-added.
  • Concurrent Modifications: If an incident is modified between the diagnostic scan and remediation PUT, etag concurrency prevents blind overwrites. Conflicting writes trigger a retry, but rapid concurrent edits may still cause transient failures.
  • Closed Incidents: Closed incidents require classification fields in the PUT body. The script preserves them from a fresh GET, but changes between GET and PUT are not merged.
  • Pagination Caps: Incident scan: 10,000 max (50 pages). Remediation candidates: 500 per run. Re-run for additional incidents.
  • Sovereign Clouds: API endpoints and versions target Azure public cloud. Government/sovereign clouds may require manual URI adjustments.
  • Rate Limiting: The script includes retry with exponential backoff for HTTP 429/5xx, but large-scale remediation (500+ incidents) may still encounter throttling.
  • PUT Semantics: Sentinel uses full-replace PUT. The script round-trips all writable properties from a fresh GET, but read-only fields are stripped. Unrecognized new fields may not be preserved.

🔍 Why Are Tags Disappearing?

Root Cause: The Sentinel Incidents – Create Or Update API uses PUT (full-replace) semantics. When any service (including the Phishing Triage Agent) updates an incident, the entire incident object is replaced. If the labels array is omitted or empty in the PUT body, all existing labels are deleted.

The Defender XDR PATCH /api/incidents endpoint behaves similarly — it overwrites the tags array rather than merging. This is by-design API behavior, not a bug. The agent is not intentionally removing tags; it simply doesn’t preserve fields it didn’t set.
