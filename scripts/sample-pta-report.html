📋 Common Reasons Incidents Are Not Processed by PTA

  • Missing URBAC / permissions — PTA requires Defender URBAC for its workload.
  • Agent identity misconfiguration — The assigned identity does not have required permissions.
  • Conditional Access policies — CA rules blocking the agent identity sign-in.
  • SCU capacity exhausted — Security Compute Unit limits reached.
  • Service or execution errors — Transient backend failures or request timeouts.
  • Alert pre-classified — Another automation classified the alert before PTA could process it.
  • Tag changes after processing — Agent indicators may be removed by downstream automations.
